The ErrorHandlerHelper

This helper is used as an error handler by using set_error_handler().

Class

Ding\Helpers\ErrorHandler\ErrorHandlerHelper

Driver

Ding\Bean\Factory\Driver\ErrorHandlerDriver

Annotations

Class level annotation @ErrorHandler

Description

PHP will call the callback registered with set_error_handler(). The property "errorHandler" of the bean, is a reference to your own bean (whose class definition must implement Ding\Helpers\ErrorHandler\IErrorHandlerDriver).

The driver is the one responsible for getting an optional bean from the container, named "ErrorHandler", that should be of class "Ding\Helpers\ErrorHandlerHelper". If this bean exists, it will be set as the error handler. The helper will in turn, call the handler set via the "errorHandler" property. This driver registers at AfterConfig in the Container lifecycle.

You need to add an option when instantiating the container:
$options['drivers'] => array('errorhandler' => array())
